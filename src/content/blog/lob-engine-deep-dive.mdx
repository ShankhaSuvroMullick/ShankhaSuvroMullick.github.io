# Building a Sub-Microsecond LOB Matching Engine in C++17

A limit order book (LOB) is the central data structure of every electronic exchange. Orders sit in the book at price levels, and the matching engine pairs incoming orders with resting ones.

The challenge: do this in under 1 microsecond.

## Architecture Overview

The engine has three components:

1. **Order ingestion** — lock-free MPSC queue, producers never block
2. **Order book** — per-side price-level arrays with SIMD search
3. **Execution engine** — atomic operations for fill matching

## The Cache Line Problem

The naive implementation puts bids and asks in adjacent memory. At high throughput, the CPU spends most time on false sharing — the cache line containing the best bid gets invalidated every time the best ask updates.

**Fix:** Align bid and ask structures to separate cache lines.

```cpp
alignas(64) struct BidSide { /* ... */ };
alignas(64) struct AskSide { /* ... */ };
```

This alone cut P99 latency from 8μs to 3μs.

## SIMD Price Search

Sequential scan for best price: ~40ns. With AVX2:

```cpp
__m256i target = _mm256_set1_epi32(price);
__m256i levels = _mm256_loadu_si256((__m256i*)price_array);
__m256i cmp = _mm256_cmpeq_epi32(levels, target);
int mask = _mm256_movemask_epi8(cmp);
```

4x speedup. Median drops below 1μs.

## Results

- **< 1μs median** order processing
- **< 5μs P99** under 2M events/sec
- **0 heap allocations** in hot path
